SPA Client PKCE

OAuth2 授权码 + PKCE 流程

此 SPA 应用使用公共客户端（client_id: spa-client），无 client_secret。
通过 PKCE (Proof Key for Code Exchange) 保护授权码流程的安全性。

PKCE 流程说明

  1. 客户端生成随机 code_verifier
  2. 计算 code_challenge = SHA-256(code_verifier)
  3. 授权请求携带 code_challenge
  4. 令牌请求携带原始 code_verifier
  5. 授权服务器验证哈希匹配，签发令牌

🔍 Token Introspection & Revocation

    

    
Introspection 通过 /oauth2/introspect (oidc-client Basic Auth)；Revocation 通过自定义 /api/revoke 端点直接操作 OAuth2AuthorizationService 吊销 token。
注意：JWT 是无状态的，吊销不会立即使 resource-server 拒绝，需本地清除 token。
